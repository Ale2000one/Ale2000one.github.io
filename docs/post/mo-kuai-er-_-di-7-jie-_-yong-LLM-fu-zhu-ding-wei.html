开场：为什么定位需要AI辅助

在上一节中，我们学习了如何通过"三层定位"框架找到学习的关键卡点。

但有一个问题始终存在：你可能不知道自己不知道什么。

这是一个著名的认知陷阱——"不知道自己不知道"（Unknown Unknowns）。

举例：

一个产品经理想学数据分析，她可能会定位为：

  • "我需要学SQL"
  • "我需要学Python"
  • "我需要学数据可视化"

但实际上，她真正卡住的是"不知道该分析什么"——她不懂业务指标的定义，不知道哪些数据是关键。

这个盲区，她自己很难发现。

AI辅助定位的价值：

AI可以帮助你跳出自己的视角，用"专家视角"审视你的定位：

  • AI可以帮你发现"你不知道自己不知道"的地方
  • AI可以模拟专家视角，指出关键卡点
  • AI可以从多个角度分析你的问题，提供多元化的定位方案

本节将教你如何利用AI进行精准定位。

第一部分：AI辅助定位的价值

1.1 AI能帮你做什么

价值一：识别"不知道自己不知道"的领域

AI基于其训练过程中学到的大量知识，能够识别出你在某个领域可能存在的盲区。

当你描述你的问题时，AI可能会指出：

  • "你提到了X，但你可能还需要了解Y"
  • "你关注了表层问题，但更核心的可能是Z"
  • "这个问题背后有一个你可能没考虑到的概念"

这些提示，能帮你发现自己忽视的盲区。

价值二：模拟专家视角，发现关键点

AI可以扮演不同领域的专家角色，用专家的视角审视你的问题。

专家和普通人的区别在于：

  • 专家知道什么是核心，什么是边缘
  • 专家知道新手常犯的错误是什么
  • 专家知道从哪里切入最容易突破

通过AI模拟专家视角，你可以获得专家级的定位建议。

价值三：提供多种定位方案的对比

AI可以针对同一个问题，提供多种不同的定位思路。

你可以：

  • 让AI从不同角度分析你的问题
  • 让AI给出多种可能的定位方案
  • 然后根据自己的情况，选择最合适的方案

1.2 AI辅助定位 vs 人工定位

维度 人工定位 AI辅助定位
视角 受限于自己的认知 可以模拟专家视角
盲区 难以发现自己的盲区 能识别"不知道不知道"
速度 需要自己思考推理 即时响应，快速迭代
深度 取决于个人经验 受限于AI知识库
准确性 依赖个人判断 需要验证

核心结论：

AI定位是人工定位的有力补充，不是替代。

用AI发现盲区，用人工最终判断。

第二部分：定位Prompt的四种技巧

2.1 技巧A：问题反推

核心思想：

描述你遇到的问题，让AI帮你分析最可能的卡点在哪里。

适用场景：

  • 你有一个具体的困境，但不知道卡在哪里
  • 你尝试了很多方法，但效果不好
  • 你感觉"卡住了"，但说不清卡在哪里

Prompt模板：

我遇到了一个学习/工作上的问题，请帮我分析最可能的卡点在哪里。

【问题描述】
[详细描述你的具体情况，包括：你在做什么、遇到了什么困难、你尝试了什么方法]

【请帮我分析】
1. 导致这个问题的核心原因可能是什么？
2. 如果我是新手，最可能犯的错误是什么？
3. 我可能忽视了哪些重要因素？

请从多个角度分析，帮助我发现我可能没看到的问题。

示例对话：

用户：我是一名运营，想学数据分析来优化活动效果。我学了两周Python，但感觉没什么进展，做活动效果分析还是不会。

AI分析：

从你的描述看，问题可能不在Python本身，而在别的地方：

  1. 定位可能偏了：你学的是Python"语法"，而不是数据分析"思维"。学两周Python语法，做数据分析还是不会，因为这两者之间的差距很大。

  2. 可能卡在业务理解：活动效果分析的前提是理解"什么算活动成功"——这叫"归因"。如果你不懂归因逻辑，学再多工具也没用。

  3. 可能缺少数据基础：Python只是工具，你还需要知道去哪里找数据、数据的结构是什么。

建议：与其继续学Python，不如先问自己两个问题：

  • "活动效果怎么看？"（业务理解）
  • "活动数据存在哪里？"（数据基础）

把这两个问题搞清楚，再决定要不要学Python、学什么程度的Python。

2.2 技巧B：专家视角

核心思想：

让AI扮演一个有多年经验的专家，告诉你这个领域最关键的是什么、新手常犯什么错误。

适用场景：

  • 你不确定自己的定位是否正确
  • 你想知道"如果我是专家，会怎么看这个问题"
  • 你想避免新手常犯的错误

Prompt模板：

请扮演一个有10年[领域名称]经验的专业人士，从专家视角帮我分析这个问题。

【我的背景】
[描述你的专业背景、目前的工作/学习状态]

【我的问题/困惑】
[描述你遇到的具体问题]

【请从专家视角回答】
1. 根据你的经验，我这个问题最可能的根源是什么？
2. 很多新手遇到类似问题时会犯什么错误？
3. 如果你来处理这个问题，你会怎么做？
4. 有没有我可能忽视的重要信息？

请像在给我建议一样，用第一人称回答。

示例对话：

用户：我是一名设计师，想学Figma做UI设计。我下载了Figma，看了很多教程，但感觉功能太多，不知道该从哪里开始。

AI（专家视角）：

作为一个做了10年UI设计的人，我看到你面临的问题其实很典型。

问题根源：你陷入了"功能清单式学习"——把所有功能都看一遍，想记住每个功能。这是效率最低的方式。

新手常犯的错误：

  1. 从菜单和功能开始学，而不是从实际页面开始学
  2. 想记住所有快捷键，其实常用快捷键只有5-6个
  3. 跳过组件和Auto Layout，这两个功能能解决80%的问题

我会怎么做：

  • 找一个你喜欢的设计作为参考
  • 用Figma复刻这个设计，只用最基础的功能
  • 遇到不会的再查，这样学的最精准

你可能忽视的：Figma的核心是"组件"和"约束"，理解了这两个，设计效率能提升10倍。

2.3 技巧C：清单核查

核心思想：

让AI列出某个领域的核心概念清单，标注哪些是必须先掌握的、哪些可以后学。

适用场景：

  • 你不确定学习一个领域需要掌握哪些内容
  • 你想知道哪些是核心、哪些是边缘
  • 你想确认自己的学习范围是否正确

Prompt模板：

请帮我列出[领域名称]中最重要的20个概念，并标注：
- 必须先掌握的（不掌握后续无法进行）
- 进阶需要掌握的（有精力再学）
- 可以后学的（需要时再查）

【我的背景】
[描述你的背景和当前水平]

【我的学习目标】
[描述你想达到的状态]

请按学习顺序排列，并说明为什么这个顺序。

示例对话：

用户：我想学数据分析用于工作，请列出最重要的20个概念并标注学习顺序。

AI回答：

【数据分析 - 20个核心概念 + 学习顺序】

第一阶段：必须先掌握（理解后再学其他）

  1. 业务指标定义 - 不懂这个，其他都是空谈。知道"活跃用户"怎么定义、"转化率"怎么算
  2. 数据表基础 - 知道数据存在哪里、表和字段是什么意思
  3. 维度与度量 - 数据分析的基本语言，理解"什么是维度、什么是度量"

第二阶段：进阶需要（边做边学）

  1. SQL基础查询 - SELECT、WHERE、GROUP BY
  2. 数据聚合与汇总 - COUNT、SUM、AVG等
  3. 时间序列分析 - 环比、同比、趋势
  4. 常见图表类型 - 折线图、柱状图、饼图的适用场景
  5. 对比分析 - 和谁比、怎么比

第三阶段：可以后学（需要时再查）

  1. SQL高级查询 - JOIN、子查询
  2. 数据清洗 - 缺失值、异常值处理
  3. 高级可视化 - 组合图表、交互图表
  4. 统计基础 - 方差、相关性
  5. Python基础 - 非必要，Excel/SQL足够

为什么这样排序：

  • 业务指标是"方向"，方向错了，后面都白学
  • 数据表基础是"地图"，不知道数据在哪，无法分析
  • SQL/Python是"工具"，工具可以边做边学

2.4 技巧D：差距分析

核心思想：

描述你的背景和目标，让AI帮你分析差距在哪里、最需要补充什么。

适用场景：

  • 你知道自己的起点和终点，但不知道中间缺什么
  • 你想知道自己和目标之间差多少
  • 你想制定精准的学习计划

Prompt模板：

我有一个学习/职业目标，请帮我分析我和目标之间的差距，以及我应该怎么弥补。

【我的背景】
- 我的专业/工作：
- 我已有的技能：
- 我目前的水平：

【我的目标】
- 我想达到的状态：
- 我想能做什么：

【请帮我分析】
1. 我和目标之间最大的差距是什么？
2. 为了达到目标，我最需要补充什么？
3. 我应该按照什么顺序学习？
4. 有没有我可能忽视的隐性要求？

请给出具体、可操作的建议。

示例对话：

用户：我是一名传统零售的产品经理，想转行做互联网产品经理，请分析我需要补什么。

AI回答：

【差距分析与学习路径】

你的优势（可以迁移的）

  • 产品基本功：需求分析、项目管理这些可以迁移
  • 业务理解：零售行业经验是有价值的
  • 沟通能力：这个是通用的

你的差距（需要补的）

  1. 互联网行业认知（优先级最高）

    • 你不了解互联网产品和用户
    • 建议：花1周了解互联网常见产品形态和用户行为

  2. 数据分析能力（中优先级）

    • 互联网产品经理每天都在和数据打交道
    • 你目前的Excel基础可以，但要补SQL和基本分析思维

  3. 互联网工作方式（中优先级）

    • 敏捷开发、A/B测试、增长黑客等概念
    • 建议：了解基本概念即可，深度在工作中学习

  4. 相关工具（低优先级）

    • Axure/Figma做原型
    • 这是工具，边做边学最快

学习顺序建议

  1. 先了解互联网行业（1周）
  2. 再补数据分析基础（2-3周）
  3. 再了解互联网工作方式（1周）
  4. 工具在实际项目中学习

可能忽视的隐性要求

  • 互联网行业变化快，学习能力比知识更重要
  • 作品集很重要：准备一个模拟项目，展示你的产品思维

第三部分：与AI对话的艺术

3.1 追问：从"这个卡点"深入到"具体原因"

AI给出的回答往往是概括性的。要获得更深入的洞察，需要追问。

追问技巧：

  1. 追问"为什么"：让AI解释判断依据

    • "你说我卡在归因逻辑上，为什么这样认为？"

  2. 追问"具体是什么"：让AI给出更具体的描述

    • "归因逻辑具体指什么？能举个例子吗？"

  3. 追问"怎么做"：让AI给出可操作的建议

    • "如果我要解决归因问题，具体该怎么做？"

示例：

用户：我是一名运营，学了两周Python但没效果。

AI：问题可能在于你学的是Python语法，而不是数据分析思维。

追问1：为什么这样认为？（追问原因）
AI：因为Python语法和数据分析师的日常工作之间有很大差距。数据分析师80%的时间在理解问题、提取数据，而不是写代码。

追问2：具体来说，我卡在哪里？（追问具体）
AI：你可能卡在两个地方：1）不知道该提取什么数据 2）不知道如何理解数据含义。Python只是工具，这两点没解决，工具用不上。

追问3：我应该怎么做？（追问做法）
AI：建议两步：1）先搞懂"活动效果怎么看"，问你的数据分析师或领导 2）带着问题学SQL，需要什么查什么。Python可以先放一放。

3.2 验证：让AI给出判断依据

AI的定位建议不一定完全准确，需要验证。

验证方法：

  1. 要求AI提供判断依据

    • "你说我卡在X，请说明你的判断依据是什么？"

  2. 交叉验证

    • 用其他渠道（搜索引擎、问同事）核实AI的说法

  3. 小范围尝试

    • 根据AI建议，先做一个小尝试，验证是否有效

  4. 保持怀疑态度

    • AI不是权威，最终判断权在你自己

3.3 交叉：让AI从不同角度分析同一个问题

同一个问题，从不同角度分析，可能得到不同的洞察。

交叉分析技巧：

  1. 让AI扮演不同角色

    • "从产品经理的角度怎么看？"
    • "从数据分析师的角度怎么看？"
    • "从一个培训师的角度，给初学者什么建议？"

  2. 让AI分析"反面"

    • "如果我定位错了，可能错在哪里？"
    • "最糟糕的情况是什么？"

  3. 让AI对比不同方案

    • "方案A和方案B各有什么优缺点？"

示例：

用户：我是一名设计师，想转行做UX设计，我定位为"学Figma + 补用户研究知识"，这个定位对吗？

AI（设计师视角）：方向没问题，Figma是必备技能，用户研究也很重要。

AI（面试官视角）：但你需要补充一点：作品集。UX设计师找工作，作品集是第一关。你有Figma技能，但没有项目作品，面试会很吃亏。

AI（培训师视角）：建议调整顺序：先做一个完整的模拟项目，再学具体工具。这样你既有作品，又有技能。

综合建议：定位大方向是对的，但需要补充"准备作品集"这个环节，并且调整学习顺序——先做项目，再学工具。

第四部分：定位结果的验证

4.1 为什么需要验证

AI给出的定位建议可能存在以下问题：

  • AI对你的具体情况了解不够全面
  • AI的知识可能过时或有偏差
  • AI可能产生"幻觉"，给出不准确的建议

因此，AI的定位结果需要验证后才能使用。

4.2 验证的三种方法

方法一：快速尝试

根据AI的定位建议，进行一个小范围的尝试，验证是否有效。

示例：

AI说："你可能卡在业务理解上，不是工具技能上。"

验证方法：

  • 先花1-2天搞清楚"活动效果的业务指标"
  • 然后看看是否比继续学Python更有帮助
  • 如果有帮助，说明AI的判断是对的

方法二：反馈收集

和熟悉你情况的人讨论AI的定位建议，听取他们的反馈。

可以问：

  • "AI说的这个问题，你觉得对吗？"
  • "你有没有看到我忽视的地方？"
  • "如果是你，你会怎么定位？"

方法三：迭代优化

定位不是一次性完成的，而是需要迭代优化。

  • 先根据AI建议形成一个初步定位
  • 在学习过程中验证这个定位是否准确
  • 根据实际反馈不断调整

4.3 定位验证的信号

定位准确的表现：

  • 学的知识马上能用上
  • 解决了卡点之后，后续内容变得清晰
  • 学习的方向感和紧迫感变强

定位不准确的表现：

  • 学了很久但进展缓慢
  • 感觉"学了很多，但不知道该用在哪里"
  • 卡点没有真正解决

遇到这些信号，说明定位可能有问题，需要重新定位。

第五部分：案例演示——用ChatGPT进行定位分析

5.1 定位任务背景

小王是一名市场专员，公司让他负责一个新媒体的运营，他想学习"内容营销"来提升工作效果。

他不确定自己该学什么、从哪里开始，于是寻求AI辅助定位。

5.2 定位过程记录

第一步：使用技巧A（问题反推）

用户：我是一名市场专员，公司让我负责新媒体运营。我感觉内容营销很重要，想学习，但不知道从哪里开始。我尝试看了很多文章，越看越迷茫。请帮我分析我的问题可能在哪里。

AI分析：

从你的描述看，你的问题可能不是"不知道学什么"，而是：

  1. 没有具体场景：你说"内容营销很重要"，但没有说清楚你的具体任务是什么。是你要做内容规划？还是内容撰写？还是活动策划？

  2. 信息过载：看了很多文章，但没有形成系统认知。碎片化信息看多了，反而更迷茫。

  3. 可能缺少目标受众思维：内容营销的核心是"对谁说什么"。你可能没想清楚目标受众是谁。

建议：先明确你的具体任务，再针对性地学习。

第二步：使用技巧B（专家视角）

用户：具体来说，我的任务是每个月做一次品牌宣传活动，需要产出内容、策划活动、分析效果。请从专家视角告诉我，新手最可能在哪里卡住。

AI（专家视角）：

作为有10年经验的市场营销人，我告诉你，新手最容易卡在这里：

  1. 不知道"好内容"是什么：很多人以为内容营销就是写文章，结果写了一堆没人看的文章。好内容的前提是"知道读者想要什么"，而不是"你想说什么"。

  2. 没有数据思维：做完活动不知道怎么分析效果，或者分析了但不知道怎么看。好内容需要数据验证，不是凭感觉。

  3. 不懂平台规则：每个平台（抖音、微信、小红书）规则不同，照搬套路往往效果不好。

你可能卡住的地方：

  • 内容定位（这篇文章是给谁看的？）
  • 效果分析（活动效果到底好不好？）

具体建议：

  • 先搞清楚你的目标用户是谁
  • 再搞清楚用什么平台
  • 最后学内容创作技巧

第六部分：本节总结

核心要点回顾

  1. AI辅助定位的价值：

    • 识别"不知道自己不知道"的盲区
    • 模拟专家视角，发现关键点
    • 提供多种定位方案的对比

  2. 定位Prompt的四种技巧：

    • 技巧A：问题反推——让AI分析卡点在哪里
    • 技巧B：专家视角——让AI扮演专家给出建议
    • 技巧C：清单核查——让AI列出核心概念和学习顺序
    • 技巧D：差距分析——让AI分析你和目标的差距

  3. 与AI对话的艺术：

    • 追问：深入到具体原因和做法
    • 验证：让AI说明判断依据，交叉验证
    • 交叉：从多个角度分析同一问题

  4. 定位结果的验证：

    • 快速尝试：用小范围行动验证
    • 反馈收集：和熟悉情况的人讨论
    • 迭代优化：定位是一个动态过程

关键认知

AI是定位的有力助手，但不是权威。

用AI发现问题、获得建议，但最终判断权在你自己。

定位需要迭代——先有一个初步定位，然后在实践中验证和调整。
