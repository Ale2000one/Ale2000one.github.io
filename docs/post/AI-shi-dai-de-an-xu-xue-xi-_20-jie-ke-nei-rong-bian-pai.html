模块一：重置学习直觉（第1-3节）

第1节：为什么学了很多却用不上

课程目标：让学员认识到"先学后用"策略的根本缺陷

核心内容：

  1. 现象剖析 - 买书报课从头学最后不了了之的典型经历；普通人学习新领域80%知识用不到；核心问题是学习策略而非努力程度

  2. "预先学习"的三大陷阱

    • 预测陷阱：你无法准确预测需要什么
    • 遗忘陷阱：学到的知识没有实践场景支撑
    • 变形陷阱：理论到实践之间的"最后一公里"从未被跨越

  3. 重新理解学习

    • 学习不是积累，而是连接
    • 知识需要场景激活
    • "差不多就行"比"完美储备"更有价值

案例：小王学Python从入门到放弃分析；某产品经理学数据分析学了3个月只用2个公式

课后练习：回忆最近一次"从头学起"经历，列出实际用到的知识点比例

第2节：按需学习是什么，不是什么

课程目标：建立对"按需学习"的正确理解，划清边界

核心内容：

  1. 按需学习的定义

    • 核心理念：因用而学，而非因学而用
    • 起点：真实的任务或问题
    • 路径：从需求出发，按图索骥

  2. 按需学习不是什么

    • 不是碎片化学习（碎片化是逃避，按需学习是主动选择）
    • 不是功利主义（短期有用，更有长期价值）
    • 不是放弃系统性（在需要系统的地方，系统学）

  3. 按需学习的适用边界

    • 适合：技能型、实践型、交叉型学习
    • 不适合：底层原理、基础理论体系构建、考试备考

  4. 按需学习与AI的协同

    • AI降低"按需"检索的成本
    • AI可以快速填充知识空白
    • AI让"先用后学"成为可能

案例：设计师学AI绘图从Midjourney实操需求出发；创业者学财务基于融资路演需要

课后练习：选择一个想学的技能，用"按需学习"视角重新设计学习路径

第3节：你的第一张地图——选定一个学习目标

课程目标：帮助学员选定一个具体的学习目标，并画出初始地图

核心内容：

  1. 如何选择"好的"学习目标

    • 有真实场景：学完能用
    • 有时间压力：有deadline逼迫快速掌握
    • 有清晰边界：知道"学成什么样算成功"
    • 难度适中：60%熟悉 + 40%陌生

  2. 初始地图绘制方法

    • 写下对目标领域的已知
    • 标注遇到的真实问题
    • 估计各部分的难度等级

  3. "最小可行目标"设定

    • 不是"学会Python"，而是"用Python处理Excel"
    • 不是"学会写作"，而是"写出一篇能发表的文章"
    • 不是"学会投资"，而是"配置一个年化5%的基金组合"

  4. 目标校准与调整 - 学习过程中如何修正目标；如何判断是否需要扩大/缩小范围

配套PDF：《学习目标选定模板》

AI应用：演示用AI快速列出某领域"最常见的10个应用场景"

课后练习：使用模板完整填写第一个学习目标及初始地图

模块二：RIDE循环拆解与实战（第4-11节）

第4节：侦察(R)——如何在几小时内看清一个陌生领域的全貌

课程目标：掌握快速侦察陌生领域的方法论

核心内容：

  1. 侦察的目的与价值

    • 建立领域全景图
    • 识别关键概念与术语
    • 发现学习路径的线索
    • 避免"不知道从哪里下手"的困境

  2. 侦察的四个维度

    • 是什么：领域的基本定义和边界
    • 做什么：核心任务和常见应用
    • 谁在做：主要参与者、专家、机构
    • 怎么学：常见学习资源和路径

  3. 侦察的具体方法

    • 信息源分级：权威文档 > 专业社区 > 搜索引擎 > 社交媒体
    • 快速阅读技巧：抓标题、抓结构、抓高频词
    • 关键问题清单：这份资料回答了我的什么问题？

  4. 侦察的深度把控

    • 侦察不是深入学习，而是"踩点"
    • 时间盒：侦察阶段不超过总学习时间的20%
    • 识别"够用了"的信号

案例：用2小时侦察"推荐系统"；用半天时间了解"跨境电商"

课后练习：选择一个陌生领域完整执行侦察，记录时间和产出

第5节：用LLM做侦察——效率提升一个量级

课程目标：掌握用大语言模型加速侦察阶段的具体方法

核心内容：

  1. 为什么AI特别适合侦察阶段

    • 知识覆盖面广
    • 能做跨语言资料整合
    • 可以模拟"高手对话"
    • 响应速度快

  2. AI侦察的四种模式

    • 模式A：领域速览 - "给我一个领域的新手入门指南"
    • 模式B：术语解释 - "解释这个领域最重要的20个术语"
    • 模式C：专家访谈 - "假设你是10年经验专家，告诉我新手常犯的错误"
    • 模式D：路径规划 - "我要在2周内入门X，请给出学习计划"

  3. AI侦察的局限与应对

    • 可能产生"幻觉"：需要交叉验证
    • 可能过于泛化：需要追问细节
    • 可能过时：需要标注时间节点

  4. AI与人工侦察的分工

    • AI做"面"，人工做"点"
    • AI给方向，人工确认路径

配套PDF：《AI侦察Prompt模板库》

案例：完整演示用ChatGPT侦察"数据可视化"的全过程

课后练习：使用PDF中的模板选择一个领域进行AI侦察练习

第6节：定位(I)——找到那个"不解决就不动"的卡点

课程目标：掌握精准定位学习焦点的能力

核心内容：

  1. 为什么"定位"比"学习"更重要

    • 精力是有限的
    • 问题有核心与边缘之分
    • 解决关键卡点往往能带动整体突破

  2. 定位的三层框架

    • 第一层：问题定位 - 具体任务是什么？卡住的是什么？
    • 第二层：知识定位 - 需要什么知识？关键瓶颈在哪？
    • 第三层：资源定位 - 哪里可以快速获取？什么形式效率最高？

  3. "不解决就不动"法则

    • 找到那个"没有它，任务就无法推进"的点
    • 把所有精力聚焦在这一个点上
    • 其他知识"边做边学"

  4. 定位错误的代价 - 案例学了大量前端知识却卡在后端接口设计上

案例：运营想学数据分析优化活动效果

  • 错误定位：学Python、统计学、数据可视化
  • 正确定位：理解"活动效果"的核心指标及归因逻辑

课后练习：针对学习目标完整执行"三层定位"

第7节：用LLM辅助定位——让AI帮你看到盲区

课程目标：掌握用AI辅助精准定位的具体方法

核心内容：

  1. AI辅助定位的价值

    • 帮助识别"不知道自己不知道"的领域
    • 模拟专家视角，发现关键点
    • 提供多种定位方案的对比

  2. 定位Prompt的四种技巧

    • 技巧A：问题反推 - "我遇到Y问题，请分析最可能的卡点"
    • 技巧B：专家视角 - "作为10年经验专家，最常忽视的关键点是什么"
    • 技巧C：清单核查 - "请列出20个概念，标注必须先掌握的"
    • 技巧D：差距分析 - "我的背景是A，目标B，最需要补充什么"

  3. 与AI对话的艺术

    • 追问：从"这个卡点"深入到"具体原因"
    • 验证：让AI给出判断依据
    • 交叉：让AI从不同角度分析同一个问题

  4. 定位结果的验证

    • 快速尝试：用最小行动验证定位是否准确
    • 反馈收集：根据实践结果调整定位
    • 迭代优化：定位是一个动态过程

课后练习：使用PDF模板进行一次完整的AI辅助定位

第8节：深钻(D)——窄而深地攻克一个知识点

课程目标：掌握"深钻"一个知识点的有效方法

核心内容：

  1. 深钻的定义与价值

    • 窄：聚焦一个具体的知识点
    • 深：直到能独立应用
    • 目标：形成"可迁移"的理解

  2. 深钻的三个层次

    • 理解层：知道"是什么"，能用自己的话复述，能举出1-2个例子
    • 应用层：知道"怎么用"，能处理标准情况，能识别边界条件
    • 迁移层：知道"什么时候用"，能处理变体情况，能教会别人

  3. 高效深钻的方法

    • 费曼技巧：以教促学
    • 变式练习：从已知推未知
    • 刻意纠错：主动找反例
    • 间隔重复：对抗遗忘曲线

  4. 深钻的边界控制

    • 警惕"过度深钻"
    • 设置"够用"标准
    • 知道什么时候该"退出来"

案例：深钻"SQL的JOIN操作" - 理解→应用→迁移完整过程

课后练习：选择一个知识卡点，用三个层次的标准检验

第9节：用LLM做深钻——把AI变成你的私人导师

课程目标：掌握用AI辅助深度学习的具体方法

核心内容：

  1. 为什么AI特别适合深钻

    • 可随时提问，不受时间限制
    • 能根据你的水平调整解释深度
    • 可以生成大量练习题
    • 能模拟不同风格的讲解

  2. 深钻Prompt的四种模式

    • 模式A：概念解析 - "用具体场景解释概念，用类比方式"
    • 模式B：知识检验 - "给我出一道关于知识点的题"
    • 模式C：纠错训练 - "我的理解是这样，请指出错误"
    • 模式D：苏格拉底式追问 - "问越来越深入的问题"

  3. AI辅导的节奏

    • 学习前：让AI给出"学习要点清单"
    • 学习中：让AI解释卡住的地方
    • 学习后：让AI出题检验

  4. 避免AI深钻的陷阱

    • 警惕"感觉懂了"
    • 动手练习 > 看AI解释
    • 定期回顾，检测遗忘

配套PDF：《AI深钻对话指南与技巧》

课后练习：使用PDF模板选择一个知识卡点进行AI深钻

第10节：执行与萃取(E)——做，然后把经验变成知识

课程目标：理解"执行"与"萃取"的完整闭环

核心内容：

  1. 为什么"做"是不可跳过的环节

    • 知识必须在实践中被激活
    • 做中学是最深刻的学
    • 执行中发现的新问题是最好的学习素材

  2. 执行阶段的四个动作

    • 启动：克服"不知道从何下手"
    • 推进：在错误中迭代
    • 验证：确认产出达到要求
    • 复盘：总结经验教训

  3. 萃取：把经验提炼成知识

    • 什么是萃取：从具体经验中提炼普适规律
    • 萃取的三个维度：
      • 操作层：具体怎么做
      • 原理层：为什么有效
      • 边界层：什么时候失效

  4. 萃取成果的形式

    • 个人知识库条目
    • 复盘文档
    • 可复用的模板/检查清单
    • 教学素材（未来教别人）

案例：完成一次用户访谈后的萃取 - 三个维度的完整展示

课后练习：完成一个实践任务并尝试萃取其中经验

第11节：用LLM辅助萃取——让AI帮你从经验中提炼精髓

课程目标：掌握用AI辅助萃取的具体方法

核心内容：

  1. 为什么萃取需要AI辅助

    • 萃取需要"跳出来看自己"
    • AI可以提供第三视角
    • AI能发现你可能忽视的模式

  2. 萃取Prompt的四种模式

    • 模式A：经验总结 - "完成X遇到Y问题，分析做得好/可改进/可提炼的规律"
    • 模式B：模式识别 - "三个项目中遇到类似问题，分析共同原因"
    • 模式C：原理挖掘 - "方法X成功了，分析有效底层原理"
    • 模式D：知识外化 - "整理成检查清单，方便未来直接使用"

  3. AI萃取的局限

    • AI无法完全理解你的具体情境
    • 萃取结果需要你验证
    • 最终判断权在你自己

  4. 萃取成果的迭代

    • 每次实践后更新萃取内容
    • 定期整理个人知识库
    • 与他人分享获得反馈

配套PDF：《AI萃取对话模板与个人知识库模板》

课后练习：使用PDF模板萃取最近的一个项目经验

模块三：真实场景的复杂性（第12-16节）

第12节：不同领域类型的策略调整

课程目标：学会根据领域特点调整RIDE循环的应用方式

核心内容：

  1. 领域分类框架

    • 硬技能领域（编程、数据分析、设计工具）：可验证、有标准答案、需要大量练习 → 更多深钻，更多执行
    • 软技能领域（沟通、管理、领导力）：难以量化、情境依赖、需要反思 → 更多萃取，更多复盘
    • 混合领域（产品设计、项目管理）：软硬兼备 → 交替进行，动态平衡

  2. 不同领域的情境案例

    • 编程：侦察技术栈全貌，定位核心语法/框架，深钻语言特性
    • 管理：侦察管理场景全览，定位团队问题诊断，深钻沟通/决策
    • 设计：侦察设计流派/工具，定位用户需求理解，深钻设计方法论

  3. 调整的信号

    • 学习进展缓慢
    • 感觉"学了很多，但用不上"
    • 反复在同一个地方卡住

  4. 常见调整策略

    • 侦察太浅 → 回到侦察，补全领域认知
    • 定位不准 → 用AI重新定位
    • 深钻不够 → 降低范围，聚焦最核心的点

案例：学"数据分析"的不同路径（技术向 vs 业务向）；学"写作"的不同策略

课后练习：分析学习目标属于哪种领域类型，调整RIDE策略

第13节：同时学多个东西——认知资源的分配策略

课程目标：掌握多任务学习场景下的资源分配方法

核心内容：

  1. 为什么需要同时学多个东西

    • 工作需要多元化能力
    • 学习兴趣的多样性
    • 时间碎片化的现实

  2. 多任务学习的挑战

    • 注意力分散
    • 记忆干扰
    • 进展缓慢

  3. 资源分配的三种策略

    • 策略A：串行优先 - 一次只学一个，直到"可用"再换下一个。适用有紧急deadline，优点专注效率高，缺点周期长
    • 策略B：并行推进 - 同时进行多个学习项目，每天只投入碎片时间。适用长期能力建设，优点保持敏感度，缺点效率低
    • 策略C：主次结合 - 一个主攻领域 + 1-2个次要领域。适用需要平衡深度与广度

  4. 多任务学习的AI辅助

    • AI帮助制定多任务计划
    • AI帮忙追踪不同领域的进展
    • AI帮你做定期的"领域切换"复盘

案例：产品经理同时学数据分析 + 设计思维 + 管理学，如何分配时间精力

课后练习：列出需要同时学习的多个领域，用三种策略之一制定计划

第14节：从碎片到结构——把按需学到的东西整合成功力

课程目标：学会把碎片化知识整合为系统性能力

核心内容：

  1. 为什么碎片化知识无法直接变成能力

    • 碎片知识是"散点"
    • 能力需要"连线"
    • 连线需要"结构"

  2. 知识整合的三个层次

    • 第一层：点→线 - 找到知识之间的关系，形成因果链或流程链
    • 第二层：线→面 - 找到知识之间的共同模式，形成方法论或框架
    • 第三层：面→体 - 建立跨领域的连接，形成个人知识体系

  3. 整合的具体方法

    • 知识卡片：每学一个知识点，写一张卡片
    • 知识图谱：把卡片连成图
    • 主题学习：围绕一个主题，串起多个卡片
    • 定期复盘：每月整理一次知识结构

  4. AI辅助整合

    • 用AI帮你发现知识之间的关系
    • 用AI帮你发现知识体系中的"空白"
    • 用AI帮你写知识文档

配套PDF：《个人知识整合工作表》

案例：运营把碎片化的"数据分析"知识整合成"数据驱动运营"能力体系

课后练习：用PDF中的工具整理最近学习的知识碎片

第15节：借力——如何利用专家和社群加速学习

课程目标：学会借助外部资源加速学习

核心内容：

  1. 为什么要借力

    • 个人时间有限
    • 专家经验宝贵
    • 社群提供反馈

  2. 借力的四种方式

    • 方式A：找专家 - 识别专家、学会提问、付费咨询的价值、AI模拟专家
    • 方式B：加入社群 - 选择高质量社群、主动贡献、寻找学习伙伴
    • 方式C：利用AI - 扮演不同角色（专家/新手/批评者）、帮你分析问题、检查学习成果
    • 方式D：混合使用 - AI+社群、AI+专家、专家+社群组合

  3. 借力的边界

    • 借力不是依赖
    • 最终要形成自己的判断力
    • AI和专家都是辅助，自己的思考才是核心

  4. AI时代的借力新方式

    • 用AI做"预学习"，带着问题去请教
    • 用AI做"笔记整理"，快速消化专家内容
    • 用AI做"讨论伙伴"，在交流中深化理解

案例：如何利用AI和社群学习"写作"（从投稿被拒到成功发表）

课后练习：设计你的"借力计划"，列出想利用的专家、社群和AI工具

第16节：当RIDE循环卡住了——诊断与修复

课程目标：学会诊断RIDE循环的问题并修复

核心内容：

  1. RIDE循环卡住的信号

    • 侦察阶段：找不到方向，越看越乱
    • 定位阶段：什么都觉得重要，定位不了
    • 深钻阶段：反复在一个点上打转，进不去
    • 执行阶段：做了很多，但感觉没进步
    • 萃取阶段：经验很多，无法提炼

  2. 卡住的诊断维度

    • 信息量：太多/太少？
    • 难度：太高/太低？
    • 目标：太模糊/太具体？
    • 节奏：太快/太慢？

  3. 常见卡住模式与修复方法

    • 侦察瘫痪（信息过载）→ 限制时间，强制输出
    • 定位失焦（问题模糊）→ 回到具体任务，重新定位
    • 深钻陷入（追求完美）→ 设定"够用"标准，强制退出
    • 执行拖延（害怕失败）→ 降低预期，从最小行动开始
    • 萃取空洞（经验不足）→ 增加执行次数，积累更多素材

  4. AI辅助诊断

    • 让AI帮你分析"卡在哪里"
    • 让AI帮你设计"下一步行动"
    • 让AI帮你追踪"修复进度"

配套PDF：《RIDE循环诊断工具与修复手册》

案例：学员在"学数据分析"过程中的卡住与修复全过程

课后练习：回顾最近一次"学不下去"的经历，用诊断工具分析原因

模块四：心智层与边界（第17-20节）

第17节：和"不知道"共处——提高学习的心理门槛

课程目标：建立健康的学习心态，与未知和平共处

核心内容：

  1. 为什么"不知道"让人痛苦

    • 教育的误区：把"不知道"视为失败
    • 成长的阻力：害怕暴露无知
    • 效率的敌人：因为恐惧而拖延

  2. 重构"不知道"的认知

    • 不知道是学习的起点，不是终点
    • 不知道代表"可以学"的可能性
    • 不知道是可以分享的状态，不是需要隐藏的缺陷

  3. 和"不知道"共处的技巧

    • 承认不知道，而不是假装知道
    • 把不知道变成一个问题，而不是一个威胁
    • 学会说"我不知道，但我可以查/学/问"

  4. AI如何改变"不知道"的心理

    • AI让"不知道"变得更容易解决
    • AI让"承认不知道"的成本降低
    • 但也要警惕：AI可能让你跳过"不知道"的成长

案例：从"不敢问"到"善于问"的转变；如何借助AI克服"不知道"的恐惧

课后练习：记录本周在工作中"承认不知道"的时刻，分析心理变化

第18节：知道自己不知道什么——元认知训练线路

课程目标：培养对自身认知状态的觉察能力

核心内容：

  1. 什么是元认知

    • 元认知 = "知道自己知道什么/不知道什么"
    • 元认知 = "知道自己是怎么学的"
    • 元认知 = "知道自己为什么学得好/学不好"

  2. 元认知在学习中的作用

    • 避免"不知道自己不知道"
    • 及时调整学习策略
    • 加速从经验中学习

  3. 元认知训练的四个维度

    • 维度A：认知盘点 - 定期回顾我现在在哪个水平？与目标/他人对比
    • 维度B：过程监控 - 学习中用什么方法？困难时为什么卡住？进步时什么策略有效？
    • 维度C：错误分析 - 识别错误、分析原因、预防策略
    • 维度D：策略调整 - 评估策略、探索新方法、迭代改进

  4. AI辅助元认知

    • AI帮你监控学习进展
    • AI帮你发现认知盲区
    • AI帮你优化学习方法

配套PDF：《元认知训练日志模板》

案例：产品经理通过元认知训练，3个月内从"数据分析小白"到"团队数据顾问"

课后练习：使用PDF模板完成一周的元认知训练记录

第19节：什么时候该停下来系统学一次

课程目标：学会判断何时需要从"按需"转向"系统"

核心内容：

  1. 按需学习的局限性

    • 碎片化可能导致知识有"洞"
    • 长期依赖可能形成"碎片依赖症"
    • 某些领域确实需要系统性基础

  2. 需要系统学习的信号

    • 信号1：反复在同一类型的知识上卡住
    • 信号2：无法跨领域迁移，学的都是"死的"
    • 信号3：进入某个领域的深水区，碎片知识不够用了
    • 信号4：需要理解底层原理才能继续进步

  3. 系统学习的正确打开方式

    • 不是从头学，而是针对"空白"学
    • 不是全量学，而是针对"需要"学
    • 不是被动学，而是带着问题学

  4. 按需与系统的动态平衡

    • 学习早期：按需为主
    • 学习中期：按需 + 补充系统
    • 学习后期：系统为主，内化整合

  5. 如何做系统学习的规划

    • 识别系统学习的边界（不要无限扩大）
    • 制定"最小系统"学习计划
    • 执行中保持"按需"视角

案例：工程师从"会用工具"到"理解原理"的学习转变

课后练习：分析当前某个学习目标，判断是否需要进行一次系统学习

第20节：按需学习的边界——以及超强效率的学习

课程目标：建立对按需学习的完整认知，知道边界在哪里

核心内容：

  1. 按需学习的边界在哪里

    • 边界1：需要深度理解原理的领域
    • 边界2：需要长期积累的能力
    • 边界3：考试型学习
    • 边界4：需要"肌肉记忆"的技能

  2. 按需学习的适用领域

    • 工作技能：快速上手，解决问题
    • 交叉领域：连接不同知识
    • 新兴领域：边做边学
    • 工具使用：学以致用

  3. 超强效率学习的核心法则

    • 法则1：任务导向 > 内容导向
    • 法则2：问题驱动 > 知识储备
    • 法则3：AI辅助 > 全靠自学
    • 法则4：持续迭代 > 一次性学完
    • 法则5：整合输出 > 被动输入

  4. 课程总结：RIDE循环的完整图景

    • 侦察：快速看清全貌
    • 定位：精准聚焦卡点
    • 深钻：窄而深地攻克
    • 执行：做中学，学中做
    • 萃取：把经验变知识

  5. 开启你的AI时代学习之旅

    • 课程回顾：最核心的三个收获
    • 下一步行动：你的第一个RIDE循环
    • 持续成长：加入学习社群，建立个人知识体系
